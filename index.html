Settings used on this example: 
$('#tablaDatos').pbTable({
  selectable: true,
  sortable:true,
  toolbar:{
    enabled:true,
    filterBox:true,
    buttons:['view', 'edit', 'delete']
  },
  pagination:{
    enabled: true,
    pageSize: 10
  },
  onView:function(){
    alert('View button was pressed');
  },
  onEdit:function(){
    alert('Edit button was pressed');
  },
  onDelete:function(){
    alert('Delete button was pressed');
  },
  onNew:function(){
    alert('New button was pressed');
  }
});

Properties

Property Data Default value Description
selectable Boolean true Flag to allow select rows in table.
sortable Boolean true Flag to allow order rows.
toolbar.enabled Boolean true Flag to show toolbox
toolbar.filterBox Boolean true Flag to allow filter data in table
toolbar.msgSinDatos String 'No se encontraron resultados.' This message will be shown when there are not results after filtering
toolbar.buttons Array de Strings ['view', 'edit', 'delete', 'new', 'print', 'receipt'] || false Action buttons list that will be shown.
pagination.enabled Boolean true Enable front-end pagination
pagination.pageSize Integer 10 Number of rows on each page
Warning! Notice that pb-table pagination do not send any request to server. Be careful when you use pagination. For performant reasons pagination is set to false by default.

Translations

You can translate pbTable by adding some extra file configurations. By defaults, pbTable offers translations to English, French and Italian, in extra than Spanish. To translate this plugin you have to inlude the extension file after call the plugin itself.

<script src="dist/pbtable-fr_FR.js"></script>

Change default values

If you use pbTable in many places, and you want to re-write defaults setting you can import the preferences you want to be used on each instance of pbTable. Just add the configuration file after pbTable core. This way allow avoid modifing pbTable code.

<script src="dist/pbtable-defaults.js"></script>
